# 结算对账功能使用指南

## 📋 功能说明

结算对账功能用于生成游戏结算对账单，将游戏流水数据整理成标准格式的对账单，供研发方核对和结算使用。

## 🚀 使用流程

### 第一步：录入游戏流水数据

有两种方式录入数据：

#### 方式一：手动录入（适合少量数据）

1. 点击 **"💰 结算对账"** 标签页
2. 点击 **"+ 新增结算记录"** 按钮
3. 填写以下必填字段：
   - **序号**：记录的顺序号（会自动递增）
   - **计费周期**：如 `2025年12月`
   - **游戏名称**：游戏的完整名称，如 `一起来修仙 (0.05折)`
   - **流水**：游戏的总流水金额
   - **充值金额**：实际充值到账的金额
   - **结算比例**：根据合同约定，通常为 `25`
4. 可选字段（如果没有可以填0）：
   - **测试费金额**：测试产生的费用
   - **代金券金额**：代金券使用的金额
   - **退款**：退款金额
   - **渠道费**：渠道费用百分比
   - **税费**：税费百分比
5. 系统会自动计算：
   - **实际结算金额** = 充值金额 - 测试费 - 代金券 - 退款
   - **结算金额** = 实际结算金额 × 结算比例
6. 点击 **"添加"** 保存记录
7. 重复以上步骤，添加所有游戏的流水数据

#### 方式二：批量导入（适合大量数据）

1. 点击 **"批量导入"** 按钮
2. 选择包含游戏流水数据的 Excel 文件
3. Excel 格式要求：
   - 第一行为表头，包含：序号、计费周期、游戏名称、流水、充值金额等字段
   - 支持中文或英文列名
   - 系统会自动识别列名并解析数据
4. 点击 **"导入"** 完成批量导入

### 第二步：生成对账单

1. 确认所有游戏流水数据已录入完成
2. 设置统计周期（可选）：
   - 在顶部日期筛选中选择开始日期和结束日期
   - 如果不设置，将使用当前月份
3. 点击 **"📄 生成对账单"** 按钮
4. 系统会生成完整的结算对账单，包含：
   - 标题和统计周期
   - 所有游戏流水记录表格
   - 合计行（自动汇总）
   - 结算金额大写
   - 付款方信息（左侧）
   - 收款方信息（右侧）

### 第三步：打印和发送

1. 在对账单页面，点击 **"打印对账单"** 按钮
2. 浏览器会打开打印预览
3. 选择打印机或保存为 PDF
4. 打印后，在对账单上盖章
5. 将盖章的对账单和发票一起寄给研发方

## 📊 字段说明

### 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 序号 | 记录的顺序号 | 1, 2, 3... |
| 计费周期 | 结算的月份 | 2025年12月 |
| 游戏名称 | 游戏的完整名称 | 一起来修仙 (0.05折) |
| 流水 | 游戏的总流水金额 | 3286873.00 |
| 充值金额 | 实际充值到账的金额 | 16434.37 |
| 结算比例 | 结算比例百分比 | 25 |

### 可选字段

| 字段 | 说明 | 默认值 |
|------|------|--------|
| 测试费金额 | 测试产生的费用 | 0 |
| 代金券金额 | 代金券使用的金额 | 0 |
| 退款 | 退款金额 | 0 |
| 渠道费 | 渠道费用百分比 | 0 |
| 税费 | 税费百分比 | 0 |

## 🧮 计算公式

系统会自动计算以下字段：

1. **实际结算金额** = 充值金额 - 测试费金额 - 代金券金额 - 退款
2. **结算金额** = 实际结算金额 × (结算比例 ÷ 100)

例如：
- 充值金额：10000 元
- 测试费：100 元
- 代金券：200 元
- 退款：50 元
- 结算比例：25%

计算过程：
- 实际结算金额 = 10000 - 100 - 200 - 50 = 9650 元
- 结算金额 = 9650 × 25% = 2412.50 元

## 💡 使用技巧

1. **批量导入**：如果研发方提供了 Excel 报表，可以直接导入，无需手动录入
2. **编辑记录**：点击记录列表中的"编辑"按钮可以修改已录入的数据
3. **删除记录**：点击"删除"按钮可以删除错误的记录
4. **日期筛选**：设置日期范围后，对账单的统计周期会自动更新
5. **打印优化**：打印时会自动隐藏操作按钮，只显示对账单内容

## ❓ 常见问题

### Q: 如何修改付款方或收款方信息？

A: 付款方和收款方信息是预设的默认值，如需修改请联系系统管理员。

### Q: 结算比例是多少？

A: 根据您与研发方签订的合同约定，通常为 25%，但具体比例以合同为准。

### Q: 可以导出 Excel 格式的对账单吗？

A: 目前支持打印和 PDF 导出，Excel 导出功能正在开发中。

### Q: 数据会丢失吗？

A: 所有数据都保存在浏览器的本地存储中，刷新页面不会丢失。建议定期使用"备份数据"功能导出备份。

### Q: 如何批量修改结算比例？

A: 目前需要逐条编辑记录。批量修改功能正在开发中。
